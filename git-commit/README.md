# Git Commit

自动化 Git 提交流程 — 分析改动、生成 conventional commit 信息、提交并可选推送到 GitHub。

## 快速开始

### 最推荐用法（自动模式）

```
提交 Git 改动
```

AI 会自动完成：分析改动 → 暂存 → 拆分（如有需要）→ 生成提交信息 → 提交 → 推送。

### 带 Emoji 提交

```
提交代码，使用 emoji
```

### 审核模式（逐步确认）

```
提交改动，使用审核模式
```

---

## 工作模式

| 模式 | 触发方式 | 适用场景 |
|------|----------|----------|
| **自动模式** | 默认，无需额外参数 | 日常开发，快速提交 |
| **审核模式** | `--review` 或说"审核模式" | 重要提交，需要精细控制 |

### 自动模式

AI 自主决策所有步骤，无需用户确认。如果不满意提交结果，可以 `git reset HEAD~1` 回滚。

### 审核模式

在暂存方式、拆分方案、提交信息、是否推送等关键节点暂停，等待用户确认后再继续。

## 使用示例

提交改动

提交代码，使用 emoji，签名

```
提交改动
```

## 参数

### 模式控制

| 参数 | 效果 |
|------|------|
| `--review` / `审核模式` | 启用审核模式 |
| `--no-all` | 跳过自动暂存 |
| `--no-untracked` | 不处理未跟踪文件 |

### 提交控制

| 参数 | 效果 |
|------|------|
| `--no-verify` | 跳过 Git hooks |
| `--amend` | 修改上一次提交（仅未推送的分支） |
| `--signoff` | 添加 Signed-off-by 行 |

### 内容控制

| 参数 | 效果 |
|------|------|
| `--emoji` / `使用 emoji` | 提交信息包含 emoji |
| `--scope <scope>` | 指定提交 scope |
| `--type <type>` | 强制指定提交类型（如 `--type fix`） |

## 提交类型速查

| 类型 | 说明 | Emoji |
|------|------|-------|
| `feat` | 新增功能 | ✨ |
| `fix` | 缺陷修复 | 🐛 |
| `docs` | 文档 | 📝 |
| `style` | 格式调整 | 🎨 |
| `refactor` | 重构 | ♻️ |
| `perf` | 性能优化 | ⚡️ |

## 智能拆分规则

自动拆分在以下情况触发：

| 拆分条件 | 阈值 |
|----------|------|
| 改动过大 | 行数 > 300 或 文件 > 20 |
| 跨模块 | 涉及 > 5 个顶层目录 |
| 混合类型 | 包含 2+ 种不同改动类型 |
| 独立功能 | 涉及 2+ 个不相关的功能模块 |

## FAQ

**Q: 自动模式会不会提交不该提交的内容？**
A: 使用 `--review` 逐步确认，或先用 `git add <path>` 手动暂存，或使用 `--no-untracked` 防止新文件被自动暂存。

**Q: 如何默认使用中文提交信息？**
A: 已默认配置为中文。如需英文，在触发时说"使用英文"。

**Q: 自动模式会跳过 Git hooks 吗？**
A: 不会，默认会执行 hooks。如需跳过，使用 `--no-verify`。

**Q: 如何查看最近的提交信息？**
A: 使用 `git log -1 --format=full` 或 `git log --oneline -5`。
