# PPTX 教程类演示 Skill

## 核心定位

把用户给出的主题、资料、笔记、文章、大纲或零散知识点，整理成可直接演示的教程类 PPTX。默认服务于课程、培训、技术分享、操作指南和知识讲解，不做空泛营销页。

输出目标：

- 让听众知道“为什么学、学什么、怎么做、如何自检”。
- 每页只讲一个核心教学点。
- 结构清晰、步骤可复现、示例能落地。
- 视觉简洁现代，明亮浅色主题，像高级技术课程网页。

## 适用场景

| 类型 | 适合内容 | 推荐页数 |
| --- | --- | --- |
| 快速教程 | 一个概念、一个工具、一个流程 | 6-8 页 |
| 系统课程 | 多个章节、完整知识体系 | 12-20 页 |
| 操作指南 | 步骤演示、环境配置、软件使用 | 8-14 页 |
| 培训材料 | 团队培训、内部规范、方法论 | 10-18 页 |
| 技术分享 | 技术方案、实践经验、工具链介绍 | 8-12 页 |

如果用户没有指定页数，默认生成 10 页左右。

## 默认模板风格

使用 `references/tutorial-template-style.md` 作为视觉和版式基准；使用 `config.yaml` 作为默认参数。除非用户指定其他风格，始终套用这套明亮技术课程网页风教程模板。

风格关键词：

- 简洁、现代、清爽、技术感、教学感。
- 明亮浅色主题，不使用厚重深色背景。
- 浅灰蓝到白色的柔和渐变背景，可加入非常淡的径向光晕。
- 主色使用深绿色或科技蓝；强调色使用青绿色、浅蓝色。
- 标题可使用从深色到主色的线性渐变文字。
- 整体感觉像高级技术课程网页，而不是传统商务 PPT。
- 页面留白充足，信息密度中等，每页只表达一个核心观点。
- 左下角固定放置品牌 Logo，默认使用 `assets/xiemuhou-wordmark-logo.svg`。

## 工作流程

### 1. 明确任务

从用户输入中识别：

- 主题和目标。
- 受众和难度。
- 输出页数和语言。
- 是否需要真实生成 `.pptx` 文件。
- 是否有指定模板、Logo、图片或输出路径。

用户只给主题时，自动补齐合理大纲；涉及事实、数据、品牌或版本信息时，不确定就说明假设或请求补充。

### 2. 重构教学路径

不要把原文逐段搬进幻灯片。先把内容改造成教学路径：

1. 封面：主题、受众、学习收益
2. 为什么要学：问题背景和收益
3. 学习地图：本次会讲哪些部分
4. 核心概念：用一句话定义
5. 操作步骤：分 3-5 步讲清楚
6. 代码或示例演示：真实或贴近真实
7. 常见误区：错误做法、后果、修正方式
8. 检查清单：让听众能自测
9. 总结复盘：3 个关键 takeaway
10. 下一步：练习、参考资料或行动建议

### 3. 设计页面

优先使用以下教程页型：

| 页型 | 用途 |
| --- | --- |
| `cover` | 封面，突出主题、受众和学习收益 |
| `learning-goals` | 学习目标，3-5 条 |
| `map` | 学习路线或章节导航 |
| `concept` | 解释核心概念 |
| `step-by-step` | 步骤教学 |
| `code` | 代码讲解，突出关键行 |
| `example` | 示例或场景演示 |
| `comparison` | 对比不同方案 |
| `pitfalls` | 常见错误和修正方式 |
| `checklist` | 检查清单 |
| `recap` | 总结和下一步 |

每页必须有明确标题。正文优先使用短句、编号、表格、流程图式结构，而不是长段落。

### 4. 品牌与页面质感

默认使用 `assets/xiemuhou-wordmark-logo.svg` 作为左下角固定品牌标识。Logo 内容围绕 `xiemuhou` 和 `blog.xmhweb.cn` 设计，不带外边框，适合自然融合进浅色背景。Logo 必须作为“背景品牌层”处理，像模板的一部分自然融入页面，而不是每页都能随手移动的普通内容元素。

品牌使用规则：

- Logo 默认放在左下角，所有页面使用同一尺寸，禁止封面页和正文页使用不同大小。
- Logo 宽度固定为页面宽度的 12%，高度按原始比例缩放；除非用户明确要求，不允许临时放大或缩小。
- Logo 横向位置要更靠近左侧边缘，默认左边距约 0 英寸；默认纵向位置约 6.62 英寸。
- 默认不要给 Logo 添加背景色、边框、底板或光晕；只保留文字 Logo 本身。
- 只有用户明确要求增强识别度时，才可添加非常浅的背景辅助层。
- Logo 周围保留安全距离，不与正文、页码、图表、代码块或卡片重叠。
- 页面内容区必须为 Logo 预留底部安全区：正文、表格、图表、代码块的最低边缘不得进入 Logo 安全区。
- 优先把 Logo 放入母版、背景层或合成到整页背景图中；如果工具不支持母版或背景合成，也必须先绘制 Logo，再绘制正文内容，并保持统一坐标和尺寸。
- 不要把 Logo 当作可自由布局的装饰元素；不要每页手工调整位置。
- Logo 色彩跟随主色，优先使用深绿色或科技蓝。
- 如果用户提供其他 Logo，则优先使用用户 Logo。

质感规则：

- 可使用非常淡的径向光晕和柔和阴影。
- 可使用轻微玻璃感白色面板，但不要过度卡片化。
- 可在顶部加入细进度条，使用蓝绿渐变。
- 标签使用圆角 pill 样式，浅色底 + 强调色文字。
- Bullet 使用蓝绿渐变圆点，不使用传统黑色圆点。

### 5. 处理代码片段

当 PPT 中包含代码：

- 使用等宽字体，优先 `Consolas`、`Cascadia Code`、`JetBrains Mono`，无法保证时使用 `Courier New`。
- 代码块使用浅灰或深色高对比背景。
- 做基础语法高亮：关键字、字符串、注释、函数名、数字使用不同颜色。
- 每页代码不超过 12 行；超过则拆页或只展示关键片段。
- 用标注、编号或强调色说明关键行，不要让听众自己找重点。

### 6. 生成 PPTX

当需要创建或编辑 `.pptx` 文件时，使用当前环境可用的 PPTX/Presentation 工具链生成文件。优先复用已有模板、资产和仓库风格。

生成后必须尽量做一次结构或可视化检查，确认：

- 每页标题可读。
- 文本没有明显溢出。
- 步骤、表格、代码块、提示框没有重叠。
- Logo 尺寸、位置在所有页面保持一致，且没有与正文重叠。
- Logo 已作为背景品牌层或母版层处理，不作为正文内容参与排版。
- 颜色对比足够。
- 代码字体和高亮可读。
- 页数和用户要求一致。
- 输出文件路径明确。

### 7. 输出给用户

完成后说明：

- PPTX 文件路径。
- 页数。
- 使用的模板风格。
- 是否做过校验。
- 需要用户补充素材时列出缺口。

## 版式约束

- 16:9 宽屏。
- 内容区域居中，最大宽度控制在页面宽度的 85%-90%。
- 页面内边距较大，保持 64-80px 等级的视觉留白。
- 底部为固定 Logo 保留安全区；正文主内容区默认不低于页面底部 1.05 英寸处结束。
- 字体优先使用 PingFang SC、Microsoft YaHei、思源黑体、Aptos、Arial；代码使用等宽字体。
- 每页最多 1 个主标题、1 个副标题区、1 个主内容区。
- 正文每页不超过 6 个要点；每个要点不超过 24 个汉字。
- 表格最多 4 列；超过 4 列时拆页。
- 不使用大段文字、低清图片、花哨动画。
- 不用过小字号塞满信息；小字不低于 16 pt。
- 图片必须清晰，截图要裁掉无关区域。
- 章节过渡页使用超大标题居中展示。
- 动效只允许轻微淡入、上移或渐显；禁止花哨动画。

## 内容质量标准

生成前自检：

- 是否从读者视角解释了“为什么”。
- 是否把抽象概念落到例子。
- 是否每页只有一个重点。
- 是否有清晰学习路径。
- 是否提供可执行的下一步。
- 是否避免空泛口号和装饰页。

## 参数速查

| 参数 | 作用 |
| --- | --- |
| `--pages <n>` | 指定页数 |
| `--audience <text>` | 指定受众 |
| `--level beginner|intermediate|advanced` | 指定难度 |
| `--style tutorial|workshop|course|guide|tech-share` | 指定教程子风格 |
| `--theme light|dark` | 指定明亮或深色主题，默认 `light` |
| `--logo <path/url>` | 指定 Logo |
| `--brand xmh|xiemuhou|谢幕后` | 指定默认品牌文字 |
| `--output <path>` | 指定输出路径 |
| `--language zh|en` | 指定语言 |

## 禁止事项

- 不把长文逐段搬到幻灯片。
- 不生成纯装饰型页面。
- 不用过小字号塞满信息。
- 不使用低清图片或模糊截图。
- 不使用花哨动画作为主要表达。
- 不在没有用户要求时使用夸张营销风格。
- 不伪造数据来源、引用或截图。
