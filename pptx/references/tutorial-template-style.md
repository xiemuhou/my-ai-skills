# 明亮技术课程网页感 PPT 模板样式

## 设计目标

让听众在演示过程中持续回答三个问题：

```text
我现在学到哪一步？
这一页要我理解什么？
我下一步该怎么做？
```

模板偏教程、课件、培训和技术分享，不偏营销。整体应像高级技术课程网页或网页幻灯片，而不是传统商务 PPT。

## 视觉系统

| 元素 | 规范 |
| --- | --- |
| 背景 | 浅灰蓝到白色的柔和渐变，避免纯白刺眼 |
| 主色 | 深绿色或科技蓝，用于标题、页眉、章节编号 |
| 强调色 | 青绿色、浅蓝色，用于当前步骤、圆点、标签、进度条 |
| 提示色 | 浅蓝或琥珀色，用于提示框和重点提醒 |
| 警示色 | 红色，只用于误区和风险提示 |
| 字体 | PingFang SC、Microsoft YaHei、思源黑体、Aptos、Arial |
| 代码字体 | Consolas、Cascadia Code、JetBrains Mono、Courier New |
| 图形 | 细线框、步骤条、pill 标签、检查框、轻量图标 |

推荐色值：

```text

背景白 #FFFFFF
浅蓝白 #F6F9FF
主标题深色 #0F172A
正文色 #1F2937
弱文本 #6B7280
深绿色 #006C32
科技蓝 #2563EB
青绿色 #599F7A
浅蓝色 #38BDF8

```

## 背景与质感

- 使用浅灰蓝到白色的柔和渐变背景。
- 可加入非常淡的径向光晕，例如右上蓝色光晕、左下绿色光晕。
- 面板使用白色或半透明白色，配合柔和阴影。
- 禁止厚重投影、花哨渐变和大面积深色背景。

## 页面组件

### 标题区

- 左上角放短标题。
- 右上角可放章节编号，例如 `02 / 10`。
- 标题不超过 24 个汉字。
- 副标题只补充任务背景或本页结论，不写长段落。
- 标题可使用从深色到主色的线性渐变文字。

### 教学提示框

用于放一句关键结论：

```text
记住：先讲为什么，再讲怎么做。
```

样式：浅蓝或浅琥珀底、深色文字、细边框。

### 品牌 Logo

默认使用 `assets/xiemuhou-wordmark-logo.svg`。

- 品牌内容以 `xiemuhou` 为主，`blog.xmhweb.cn` 作为博客宣传副标。
- 默认 Logo 不带外边框或底色块，直接融合进页面背景。
- 默认固定在左下角。
- 所有页面使用同一尺寸，禁止封面页和正文页使用不同大小。
- Logo 宽度固定为页面宽度的 12%，高度按原始比例缩放。
- Logo 横向位置更靠近左侧边缘，默认左边距约 0 英寸；默认纵向位置约 6.62 英寸。
- Logo 需要与页面主色融合，像背景品牌层，不要像外贴水印。
- 默认不要给 Logo 添加背景色、边框、底板或光晕；只保留文字 Logo 本身。
- 只有用户明确要求增强识别度时，才可添加非常浅的背景辅助层。
- Logo 周围保留安全距离，不能与正文、页码、代码块、卡片、表格或图表重叠。
- 底部必须预留 Logo 安全区，正文主内容区不得压到 Logo 上方。
- 优先使用母版、背景层或合成进整页背景图；如果工具不支持，先绘制 Logo，再绘制正文内容，并在所有页面复用同一坐标和尺寸。
- 不要把 Logo 当作每页可自由移动的普通装饰元素。

### 顶部进度条

- 顶部放 4-6px 细进度条。
- 使用蓝绿渐变，例如 `#60A5FA -> #34D399`。
- 进度条只表达课程进度，不要抢主体内容。

### 步骤条

用于 3-5 步流程：

```text
1 输入材料 -> 2 重构大纲 -> 3 设计页面 -> 4 生成 PPTX -> 5 校验
```

当前步骤使用青绿色强调，未完成步骤使用灰色。

### Bullet

- 不使用传统黑点。
- 使用蓝绿渐变小圆点。
- 每页最多 6 个 bullet。
- 每个 bullet 用短句，不写长段落。

### Pill 标签

- 用于课程阶段、关键词、难度、工具名。
- 形状为大圆角 pill。
- 样式为浅色底 + 强调色文字。
- 不要把正文全部做成标签。

### 代码块

用于解释命令、配置或程序逻辑：

```text
const lesson = createDeck({
  style: "modern-light-tech",
  audience: "beginner",
});
```

样式：深色代码背景、等宽字体、基础语法高亮。每页不超过 12 行代码。

### 对比表

用于解释选择：

| A | B | 何时选 |
| --- | --- | --- |
| 快速教程 | 系统课程 | 时间短、目标单一时选快速教程 |

### 检查清单

用于收尾：

```text
[ ] 每页只有一个重点
[ ] 例子能落地
[ ] 操作步骤可复现
```

## 推荐页型

| 页型 | 内容结构 |
| --- | --- |
| cover | 标题、受众、学习收益 |
| learning-goals | 3-5 个目标 |
| map | 章节路线 |
| concept | 一句话定义、解释、例子 |
| step-by-step | 步骤编号、操作说明、注意点 |
| code | 代码片段、关键行标注、解释 |
| example | 场景、操作、结果 |
| comparison | 选项、优缺点、建议 |
| pitfalls | 错误做法、后果、修正方式 |
| checklist | 可执行检查项 |
| recap | 3 个 takeaway 和下一步 |

## 排版规则

- 16:9。
- 正文建议 22-28 pt。
- 小字不低于 16 pt。
- 每页最多 6 条 bullet。
- 每页只放一个核心观点。
- 内容区域居中，最大宽度约为页面宽度的 85%-90%。
- 页面内边距较大，保持 64-80px 等级的视觉留白。
- 底部为固定 Logo 预留安全区；正文、表格、图表、代码块不要进入该区域。
- 表格最多 4 列，超过就拆页。
- 图片和截图必须清晰，裁掉无关区域。
- 禁止大段文字、低清图片和花哨动画。
- 章节过渡页使用超大标题居中展示。

## 动效

- 只使用轻微淡入、上移、渐显。
- 禁止旋转、弹跳、复杂路径动画。
- 动效是辅助节奏，不承担内容表达。

## 质量检查

生成后检查：

- 标题是否清楚。
- 是否存在长段落。
- 教学路径是否顺。
- 是否有示例、代码或复盘。
- 文字是否溢出。
- 元素是否重叠。
- 代码块是否可读。
- Logo 是否清楚但不干扰内容。
- Logo 是否所有页面尺寸一致。
- Logo 是否已作为背景品牌层或母版层处理。
- 正文是否避开 Logo 安全区。
- 页面是否像技术课程网页，而不是传统商务 PPT。
