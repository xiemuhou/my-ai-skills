# better-prompt — 高质量 Prompt 重构

本 README 面向使用者：如何触发并正确使用 `better-prompt` skill。执行细节和默认参数在 `SKILL.md` 与 `config.yaml`。

`better-prompt` 用来把简单、模糊或零散的提示词，重构成更清晰、更具体、更可执行的高质量 prompt。默认只优化 prompt，不直接执行 prompt 里的任务。

## 快速开始

```text
请使用 better-prompt skill 优化下面这个提示词
输入：我想生成一个 PPT skill
输出：一个可直接复制使用的高质量 prompt
```

只想要最终 prompt：

```text
只输出优化后的 prompt，不要解释：
我想生成一个 PPT skill
```

## 触发方式

| 你的需求 | 推荐说法 |
| --- | --- |
| 优化提示词 | `优化这个 prompt：...` |
| 改写模糊需求 | `把这句话改成高质量 prompt：...` |
| 生成可复用模板 | `把这个 prompt 改成模板，缺失信息用占位符` |
| 面向 Codex | `帮我改成适合 Codex 执行的 prompt` |
| 只要结果 | `只输出优化后的 prompt，不要解释` |
| 英文触发 | `better-prompt: ...` |

## 默认输出

```markdown
## 优化后的 Prompt

```text
<可直接复制使用的 prompt>
```

## 改进点

- <说明补充了什么>
- <说明为什么更可执行>

## 使用建议

<可选建议>
```

## 优化重点

| 维度 | 会补强什么 |
| --- | --- |
| 角色 | 明确 AI 应该以什么身份完成任务 |
| 背景 | 补齐目标、受众、使用场景 |
| 任务 | 把模糊目标拆成可执行步骤 |
| 约束 | 写清风格、范围、禁止事项 |
| 输入 | 明确用户需要提供什么材料 |
| 输出 | 指定格式、语言、长度或文件结构 |
| 质量标准 | 给出可检查的验收条件 |

## 示例

原始 prompt：

```text
帮我写一个 skill
```

优化后会变成类似：

```text
# Role
你是一位资深 Codex Skill 设计师，熟悉 Agent Skills 的结构、触发机制和用户文档写法。

# Task
请帮我设计一个名为 <skill-name> 的 Codex Skill。

# Requirements
- 生成完整的 SKILL.md
- 包含触发条件、工作流程、输出规范、质量检查和禁止事项
- 如果需要用户文档，同时生成 README.md
- 文件内容使用中文

# Output Format
请输出推荐目录结构和每个文件的完整内容。
```

## 注意事项

- 默认不执行优化后的 prompt。
- 信息不足时，会优先使用 `<占位符>`，而不是随意猜测。
- 高风险或方向明显不清楚的任务，可能会先问少量问题。
- 默认输出中文，除非你要求英文。

## 常见问题

### 为什么优化后比原 prompt 长？

好的 prompt 需要把任务、边界和输出格式说清楚。长度不是目的，可执行性才是目的。

### 可以只输出 prompt 吗？

可以，直接说：

```text
只输出优化后的 prompt，不要解释
```

### 它会顺便帮我完成任务吗？

默认不会。你可以明确说“优化后顺便执行”，它才会继续做 prompt 中的任务。
