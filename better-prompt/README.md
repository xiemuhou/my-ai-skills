# better-prompt

`better-prompt` 用来把简单、模糊或零散的提示词，重构成更清晰、更具体、更可执行的高质量 prompt。

它适合用在你已经有一个大概想法，但不知道怎么把需求讲清楚的时候。

## 快速开始

```text
请使用 better-prompt 优化这个提示词：
帮我写一篇关于 AI 自动化的文章
```

输出会包含：

- 优化后的 prompt
- 改进点说明
- 使用建议

## 常用说法

| 你的需求 | 推荐说法 |
| --- | --- |
| 优化提示词 | `帮我优化这个 prompt：...` |
| 重构简单需求 | `把这句话改成高质量 prompt：...` |
| 只要结果 | `只输出优化后的 prompt，不要解释：...` |
| 面向某个模型 | `帮我改成适合 Codex 使用的 prompt：...` |
| 做成模板 | `把这个 prompt 改成可复用模板，缺失信息用占位符` |

## 默认输出格式

```markdown
## 优化后的 Prompt

```text
...
```

## 改进点

- ...

## 使用建议

...
```

## 优化重点

- 明确角色和任务目标
- 补齐上下文和输入材料
- 拆解执行步骤
- 定义输出格式
- 加入边界、约束和质量标准
- 对缺失信息使用占位符

## 示例

原始 prompt：

```text
帮我做一个网站
```

优化后：

```text
# Role
你是一位资深前端产品设计师和工程师。

# Context
我想制作一个用于展示 <品牌/产品/个人作品> 的网站，目标用户是 <目标用户>。

# Task
请为我设计并实现一个可运行的网站首页。

# Requirements
- 风格：简洁现代，重点突出核心内容
- 首屏必须清楚展示网站主题
- 包含导航、主体内容、行动按钮和基础响应式适配
- 不要做只有营销文案的空页面，页面需要有真实可用的内容结构

# Output Format
请输出实现方案，并在需要写代码时给出完整文件结构和关键代码。
```

## 注意事项

- 默认只优化 prompt，不直接执行 prompt 里的任务。
- 如果信息不足，会优先使用 `<占位符>` 保持 prompt 可复用。
- 如果你只想要最终 prompt，可以明确说“只输出 prompt”。
